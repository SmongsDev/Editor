Visual Text Editor 상세 사용 설명서 (보충)
개발 환경 및 빌드 정보
Visual Text Editor는 크로스 플랫폼 개발을 지원하며, Windows와 Linux 환경에서 모두 사용할 수 있도록 설계되었습니다.
컴파일러
GCC(GNU Compiler Collection)를 사용하여 컴파일됩니다.
플랫폼별 라이브러리
Windows: PDCurses 라이브러리 사용
Linux: ncurses 라이브러리 사용
빌드 옵션
컴파일 시 -Wall 옵션을 사용하여 모든 경고를 표시합니다.
디버깅을 위해 -g 옵션이 포함되어 있습니다.
플랫폼 감지
Makefile에서 운영 체제를 자동으로 감지하여 적절한 설정을 적용합니다.
실행 파일
Windows: viva.exe
Linux: viva
빌드 및 실행 방법
빌드하기
터미널에서 프로젝트 디렉토리로 이동한 후 make 명령어를 실행합니다.
실행하기
Windows: .\viva.exe [파일명]
Linux: ./viva [파일명]
정리하기
make clean 명령어를 사용하여 빌드된 파일들을 정리할 수 있습니다.
주의사항
Windows 환경
PDCurses 라이브러리가 필요합니다. Makefile에서 자동으로 DLL 파일을 복사합니다.
라이브러리 파일(pdcurses.dll)이 실행 파일과 같은 디렉토리에 있어야 합니다.
Linux 환경
ncurses 라이브러리가 시스템에 설치되어 있어야 합니다.
대부분의 Linux 배포판에는 기본적으로 설치되어 있지만, 없는 경우 패키지 관리자를 통해 설치해야 합니다.
2. 주요 기능
- 기본 텍스트 편집 (입력, 삭제, 줄 바꾸기)
- 파일 열기 및 저장
- 커서 이동
- 검색 기능

3. 단축키
- Ctrl+S : 파일 저장
- Ctrl+Q : 프로그램 종료
- Ctrl+F : 검색 모드 활성화
- 화살표 키 : 커서 이동
- Home/End : 줄의 시작/끝으로 이동
- Page Up/Down : 페이지 단위로 이동

4. 사용 방법

4.1 프로그램 실행
- Windows: .\viva.exe [파일명]
- Linux: ./viva [파일명]

4.2 텍스트 편집
- 일반적인 키보드 입력으로 텍스트를 입력합니다.
- Backspace로 문자를 삭제합니다.
- Enter로 새 줄을 삽입합니다.

4.3 파일 저장
- Ctrl+S를 눌러 현재 파일을 저장합니다.
- 파일명이 지정되지 않은 경우, 저장할 파일명을 입력하라는 프롬프트가 표시됩니다.

4.4 검색 기능
- Ctrl+F를 눌러 검색 모드를 활성화합니다.
- 검색어를 입력한 후 Enter를 누릅니다.
- 검색 모드에서:
  - 오른쪽 화살표: 다음 검색 결과로 이동
  - 왼쪽 화살표: 이전 검색 결과로 이동
  - Enter : 검색 모드 종료
  - ESC : 검색 모드 종료 ( 커서가 검색 모드 활성화 되기 전의 위치로 돌아감 )

6. 화면 구성
- 주 편집 영역: 텍스트를 입력하고 편집하는 공간
- 상태 바: 파일명, 총 줄 수, 현재 커서 위치 등 정보 표시
- 메시지 바: 도움말과 상태 메시지 표시

7. 기술적 특징
- ncurses/PDCurses 라이브러리 사용
- 링크드 리스트 구조로 텍스트 관리
- 동적 메모리 할당

8. 빌드 정보
- 컴파일러: GCC
- 빌드 명령어: make
- 정리 명령어: make clean
